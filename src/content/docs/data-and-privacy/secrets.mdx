Agent settings → **Secrets** → add a name/value pair. Each secret is stored encrypted, never displayed in plain text after saving.

At agent startup, secrets are injected as environment variables directly into the agent process. They never appear in logs, workspace files, or conversation history.

<Callout type="tip">
Unlike `.env` files, secrets are never on disk in plain text — they exist as environment variables only for the duration of the agent process.
</Callout>

## Scoping

Secrets are per-agent. An agent can only read its own secrets — no shared store across agents.

## Rotating a key

Agent settings → **Secrets** → update the value → restart the agent.
