## Dashboard won't load

- Check the terminal for errors — especially port conflicts
- Port in use → stop the other process or use `--port <number>`
- `node --version` must be 20+
- Cloned the repo? Run `npm install` first
- Page looks broken → hard refresh `Cmd+Shift+R`

## Agent reply never streams

- [Settings](/getting-started/configure-llm-provider) → confirm a provider is configured with a valid API key
- Check your provider's dashboard for quota or rate limit issues
- Verify the model name matches your provider's docs exactly
- Check the agent card status — start the agent if it's stopped

## Browser session won't log in

- CAPTCHA → open the [live session view](/agent-superpowers/browse-control), complete it manually
- Expired cookies → walk the agent through login again
- Profile corrupted → delete it in agent settings, log in fresh
- Chrome not found → check Chrome is installed

## Telegram doesn't respond

- Check the bot token is saved in agent settings → [Telegram](/agent-superpowers/telegram-integration)
- If webhook shows an error, save the token again to re-register
- Confirm the right agent has the token
- Agent process must be running — messages queue if it's stopped

## [Vault](/agent-superpowers/vault-obsidian-memory) is empty after restart

- Verify the workspace directory path in agent settings
- Changed the agent's ID? The vault is in the old directory — move files manually
- `GRANCLAW_HOME` mismatch → agent is looking in the wrong base directory

## Schedule didn't fire

- Check cron syntax in settings (field order: minute hour day month weekday)
- Server not running at fire time → runs at the next matching time on restart
- Server deployment? Adjust cron for the server's timezone

## High token cost

- Check cache hit rate in the [Usage panel](/using-granclaw/usage-and-costs) — low hit rate means you're paying full price
- Trim the agent's system prompt — every token there is charged on every message
- Switch to a smaller model for simple tasks

## "Capability denied" error

- Guardian enabled → open the approval queue and approve the capability
- [MCP](/agent-superpowers/mcp-tools-and-capability-approval) server not installed → agent settings → Tools → install it
- High-risk capability → requires manual approval from the guardian queue
