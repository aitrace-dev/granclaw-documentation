## Is my data sent anywhere?

Only LLM API calls leave your machine. No telemetry, no analytics, no GranClaw backend.

## Do I need internet?

For LLM calls and browser tasks, yes. The dashboard, history, vault, and settings work offline.

## Can I use GranClaw without an API key?

No. The agent needs an LLM to reply. You can explore the dashboard without one, but add a key in Settings before sending a message.

## Which LLM is best?

- **Claude** — writing, reasoning, long context
- **GPT-4o** — broad general use
- **Gemini** — Google ecosystem
- **Groq** — fast, lower cost for simpler tasks
- **OpenRouter** — one key, many models

Pick one, try it, switch if needed.

## Can two agents share a browser profile?

No. Each agent has its own isolated [Chrome profile](/agent-superpowers/browser-profiles) by design.

## Where is my data stored?

`~/.granclaw/` by default. Override with `--home <path>` or `GRANCLAW_HOME`.

- Workspaces: `~/.granclaw/workspaces/<agent-id>/`
- Databases: `~/.granclaw/data/`

## How do I back up?

[Export](/data-and-privacy/export-and-import-agents) each agent from the dashboard (vault + history + browser profile + config in one zip). Or back up `~/.granclaw/` with any file backup tool.

## Can I run it on a server?

Yes. Node.js process — runs anywhere Node 20+ runs. Browser automation needs a display or virtual framebuffer for Chrome.

## Does it work on Windows?

Not yet. macOS and Linux only. Windows support is planned.

## Is GranClaw open source?

Yes, MIT license. [github.com/aitrace-dev/granclaw](https://github.com/aitrace-dev/granclaw)

## How do I uninstall?

Delete `~/.granclaw/` (or your `GRANCLAW_HOME`). Nothing else to remove — `npx` doesn't leave a persistent install.

## Is there a mobile app?

No. The dashboard is browser-based at `localhost`. Use [Telegram integration](/agent-superpowers/telegram-integration) for mobile access.
