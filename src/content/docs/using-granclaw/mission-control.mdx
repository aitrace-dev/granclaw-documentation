Every agent has a built-in kanban. No setup. Say "plan a LinkedIn launch week" and cards appear, move through states, and report back.

![Mission Control kanban board with LinkedIn content tasks](/images/tasks-board.png)

## Card states

| State | Meaning |
|---|---|
| **Todo** | Identified, not started |
| **Doing** | Agent is working on this now |
| **Blocked** | Waiting on your input or a dependency |
| **Done** | Complete |

Both you and the agent can move cards. The agent reads the board state at the start of each turn — if you reorganize things, it adapts.

<Callout type="note">
Mission Control is per-agent. Boards never share cards between agents.
</Callout>
